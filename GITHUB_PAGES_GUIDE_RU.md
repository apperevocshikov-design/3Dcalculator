# Руководство: Как выгрузить калькулятор на GitHub и запустить сайт

Этот проект совмещает в себе мощный клиентский интерфейс (React + Tailwind CSS) и полноценный веб-сервер (Node.js + Express). 

Ниже описаны два варианта публикации:
1. **Вариант А: Быстрый запуск только клиентской части на GitHub Pages (Бесплатно)** — серверные функции будут автоматически заменены стабильной и быстрой офлайн-работой через `localStorage`.
2. **Вариант Б: Полный запуск всего приложения (Клиент + Сервер)** на специализированных хостингах (Render / Railway) с привязкой к репозиторию на GitHub.

---

## Шаг 1: Создание репозитория на GitHub

1. Зайдите на сайт [GitHub](https://github.com/) и создайте новый публичный репозиторий (например, c именем `3d-print-calc`).
2. Оставьте галочки "Add a README file" и "Add .gitignore" **выключенными**.
3. На вашем компьютере (в папке с проектом) инициализируйте git и свяжите его с вашей учетной записью:

```bash
# Инициализация Git в папке проекта
git init

# Добавление всех файлов в отслеживание
git add .

# Создание первого сохранения (коммита)
git commit -m "feat: initial commit with 3d print calculator and backend"

# Переименование главной ветки в main (стандарт GitHub)
git branch -M main

# Добавление адреса вашего созданного удаленного репозитория GitHub
# (Замените ссылку на адрес вашего репозитория)
git remote add origin https://github.com/ВАШ_ЛОГИН/3d-print-calc.git

# Отправка файлов на GitHub
git push -u origin main
```

## Вариант А: Страница сайта на GitHub Pages (Только клиентская часть)

GitHub Pages умеет бесплатно показывать только статические сайты (HTML, CSS, JS). Если вы развернете калькулятор туда, Express бэкенд не будет работать на серверах GitHub, но **наше приложение автоматически переключится в автономный офлайн-режим ("Локально")** и сохранит все ваши настройки, принтеры и пластики прямо в браузере пользователя!

### Способ 1: Сборка и деплой через инструмент `gh-pages` (Самый простой способ)

1. Откройте файл `package.json` и добавьте в него строчку `"homepage"` перед `"dependencies"`:
   ```json
   "homepage": "https://ВАШ_ЛОГИН.github.io/3d-print-calc",
   ```
2. В терминале установите пакет утилиты деплоя:
   ```bash
   npm install gh-pages --save-dev
   ```
3. Откройте `package.json` и добавьте две дополнительные команды в раздел `"scripts"`:
   ```json
   "scripts": {
     "predeploy": "npm run build",
     "deploy": "gh-pages -d dist",
     ...
   }
   ```
4. Теперь, чтобы собрать сайт и выгрузить его на живую ссылку GitHub Pages, напишите в терминале:
   ```bash
   npm run deploy
   ```
   *Утилита соберет проект в папку `dist` и сама опубликует готовую сборку в специальной скрытой ветке `gh-pages` у вас на GitHub.*
5. Перейдите во вкладку **Settings** -> **Pages** вашего репозитория на GitHub и убедитесь, что в качестве источника выбран `Deploy from a branch` с веткой `gh-pages` `/ (root)`.
6. Через 1-2 минуты ваш сайт будет доступен по ссылке: `https://ВАШ_ЛОГИН.github.io/3d-print-calc/`!

## Вариант Б: Полный запуск с сервером бэкенда (Render / Railway)

Если вам критически важно хранить общую базу принтеров, настройки и историю расчетов на вечном сервере в «облаке», чтобы база не стиралась при очистке кэша браузера, выполните эти действия:

### Размещение на бесплатном сервисе Render.com:

1. Зарегистрируйтесь на сайте [Render.com](https://render.com/) с помощью учетной записи GitHub.
2. Нажмите синюю кнопку **New+** -> **Web Service**.
3. Выберите ваш репозиторий `3d-print-calc` из списка подключенных репозиториев GitHub.
4. Настройте параметры службы:
   * **Language**: `Node`
   * **Region**: Выберите ближайший к вам (Frankfurt / Singapore)
   * **Branch**: `main`
   * **Build Command**: `npm install && npm run build`
   * **Start Command**: `npm run start`
   * **Instance Type**: `Free` (Бесплатный тариф)
5. Нажмите кнопку **Create Web Service**.
6. Render начнет установку, соберет React-клиент и запустит Express-сервер на Node.js!
7. По окончании процесса в самом верху страницы появится выданная вам постоянная веб-ссылка вида `https://3d-print-calc-xxxx.onrender.com`.

*Ваш калькулятор будет работать во всеоружии: с базой данных в `db.json` на бэкенде, полной синхронизацией и 3D-просмотром STL-моделей!*
